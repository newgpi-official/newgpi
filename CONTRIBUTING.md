# Contributing to NewGPI Documentation

Thank you for helping improve the public NewGPI documentation.

## Good ways to contribute

- Suggest a technical topic that needs a clearer explanation.
- Report an unclear statement, broken link or terminology issue.
- Propose an architecture question or realistic gaming workflow.
- Identify where a conceptual statement could be mistaken for a released capability.

## Before opening an issue

Search existing issues first. Keep each issue focused on one question or improvement.

Do not include:

- passwords, API keys, wallet keys or authentication cookies;
- personal, financial or confidential business information;
- private vulnerability details;
- unsupported claims about investment returns, asset value or product availability.

## Documentation principles

Contributions should:

- distinguish conceptual architecture from released products;
- use verifiable language;
- explain permissions, validation and governance where relevant;
- avoid financial-return promises and artificial engagement;
- link to the official website only at https://newgpi.vip.

## Suggest a topic

Use the documentation-topic issue template and explain:

1. Who needs the documentation?
2. What question should it answer?
3. Which layer is involved: experience, control, execution, data or governance?
4. What would make the answer useful and verifiable?
