# NewGPI Platform Overview

Status: Public conceptual overview

## Purpose

NewGPI is developing an infrastructure platform for AI-powered digital gaming systems. The platform vision connects intelligent agents, cloud execution, user networks and structured digital asset operations through a coordinated technology layer.

This document describes the public platform model. It does not constitute a promise that every component is generally available today.

## The infrastructure problem

An AI model can generate a decision or response, but a production gaming system must also connect users, manage context, call permitted tools, execute workloads, observe failures and maintain reliable records.

NewGPI organizes these requirements into four infrastructure pillars.

## Platform pillars

### Home Network

The connectivity layer between users, devices and digital services. Its purpose is to support dependable access across changing real-world network conditions.

### AI Orchestration

The coordination layer for agents, models, tools, context and policies. It is intended to make multi-step intelligent workflows observable and controllable.

### Cloud Infrastructure

The execution layer for scalable workloads and supporting services. It provides the computing foundation required by AI-assisted experiences.

### Digital Asset Management

The structured operations layer for digital identities, permissions, entitlements and other forms of digital value. Public implementation details will be documented before related developer features are released.

## Intended users

- Game and interactive experience developers
- AI agent builders
- Platform and cloud engineering teams
- Technology partners integrating connected digital services

## Current public status

NewGPI is publishing its platform model and technical direction. Public APIs, SDK packages and interactive demos will be documented only when they are available for external use.

## Principles

- Infrastructure before hype
- Observable and controlled AI workflows
- Clear separation between public interfaces and internal systems
- Security and privacy considered throughout the system lifecycle
- Documentation that distinguishes released capabilities from planned work
